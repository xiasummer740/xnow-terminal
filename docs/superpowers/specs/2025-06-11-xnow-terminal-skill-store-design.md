# XNOW Terminal 技能商店设计文档

## 概述

为 XNOW Terminal 的 AI Agent 添加「技能商店」功能，让小白用户也能通过点选安装来扩展 AI 能力，同时支持 AI 在后台自动总结可复用模式、生成新技能草案。

## 核心原则

1. **小白友好** — 安装流程跟手机 App Store 一样简单
2. **安全优先** — 每个技能有签名验证，安装时展示权限列表
3. **渐进增强** — 内置技能开箱即用，云端技能可选安装，AI 生成技能弹窗确认

## 系统架构

```
┌─────────────────┐     ┌────────────────────┐     ┌─────────────────┐
│   AI Agent      │────▶│   技能管理器        │────▶│   技能执行器     │
│  (对话/自动学习) │     │ (注册/存储/签名验证) │     │ (Tool Calls 运行) │
└─────────────────┘     └─────────┬──────────┘     └─────────────────┘
                                  ↕
                         ┌────────────────────┐
                         │   云端市场          │
                         │ (GitHub 仓库/社区)   │
                         └────────────────────┘
```

### 数据流

1. AI Agent 完成任务 → autoLearn 检测到可复用模式 → 生成技能草案
2. 技能管理器对草案进行签名验证 → 存入 localStorage
3. 弹出安装确认窗口 → 用户点「加入技能库」
4. 下次 AI 对话时加载已安装技能的 prompt 和 tools
5. 云端市场通过 Git 仓库同步技能列表，安装时下载对应 JSON

## 技能数据格式

```json
{
  "name": "一键备份",
  "id": "xnow-skill-backup",
  "version": "1.0.0",
  "author": "xnow-team",
  "category": "运维工具",
  "description": "一键打包备份服务器关键目录",
  "prompt": "你是一个备份专家，按以下步骤操作...",
  "tools": [
    {
      "name": "backup_directory",
      "description": "备份指定目录为 tar.gz",
      "parameters": {
        "type": "object",
        "properties": {
          "path": { "type": "string", "description": "要备份的目录路径" }
        }
      },
      "permissions": ["exec:cmd", "fs:read", "fs:write"]
    }
  ],
  "signature": "MEUCIQDGK...",
  "source": "builtin | market | ai_generated"
}
```

### 字段说明

| 字段 | 说明 |
|------|------|
| `name` | 技能显示名称 |
| `id` | 唯一标识符 |
| `version` | 语义化版本号 |
| `author` | 作者标识 |
| `category` | 分类（运维/监控/部署/安全/文件/AI） |
| `description` | 技能简介 |
| `prompt` | 注入到 AI system prompt 的指令文本 |
| `tools` | 该技能提供的工具定义（兼容 OpenAI Tool Call 格式） |
| `tools[].permissions` | 该工具需要的权限列表 |
| `signature` | 数字签名（防止篡改） |
| `source` | 来源：`builtin` 内置 / `market` 市场 / `ai_generated` AI生成 |

### 安装确认弹窗

点击「安装」后弹出模态框，展示：
- 技能名称、版本、作者、签名状态
- 技能描述
- 权限列表（每个权限用 🔑 图标 + 文字说明）
- 技能代码预览（JSON 格式）
- [取消] / [确认安装] 按钮

### AI 自动发现弹窗

AI 完成任务后生成新技能时弹出的通知：
- 🧠 图标 + "AI 学会了一个新技能！"
- 技能名称和简短描述
- [✓ 加入技能库] / [看看再说] / [忽略] 三个按钮

## 存储方案

使用 `localStorage`，与现有记忆系统保持一致：

| Key | 说明 |
|-----|------|
| `xnow_skills_index` | 已安装技能列表（只存 id + name + version） |
| `xnow_skill_{id}` | 每个技能的完整 JSON 数据 |
| `xnow_market_cache` | 云端市场技能列表缓存 |

技能加载时机：AI Agent 每次启动时从 localStorage 加载已安装技能的 prompt 和 tools，合并到 system prompt 中。

## 安全机制

### 签名验证

1. **内置技能**：随 APP 发布，带 XNOW 团队签名
2. **云端技能**：从市场下载时验证签名，无签名或签名不匹配弹警告
3. **AI 生成技能**：无签名，但安装时明确标注「AI 生成，未经验证」

### 权限控制

每个工具声明 `permissions` 字段，安装时展示给用户。权限分为：

| 权限 | 说明 | 风险等级 |
|------|------|---------|
| `exec:cmd` | 执行 shell 命令 | 🔴 高 |
| `fs:read` | 读取本地文件 | 🟡 中 |
| `fs:write` | 写入本地文件 | 🔴 高 |
| `network:fetch` | 发起网络请求 | 🟡 中 |
| `terminal:control` | 控制终端标签页 | 🟢 低 |

高风险权限在安装时用红色标注。

## 云端市场同步机制

1. XNOW 维护一个 GitHub 仓库作为技能市场源：`xnow-skills`
2. 技能商店首次打开时从仓库拉取 `index.json` 获取技能列表
3. 用户安装技能时下载对应 `skills/{id}.json`
4. 定期检查更新（每天一次），有新版技能时标记「可更新」
5. 社区用户通过 PR 提交技能到仓库

## 内置技能列表（初版）

| 技能 | 说明 |
|------|------|
| 一键备份 | SSH 连接后一键打包备份关键目录 |
| 日志分析 | 扫描 Nginx/Apache 日志统计 IP、状态码 |
| 网站监控 | 定时检测网站 HTTP 状态和响应时间 |
| 批量部署 | 向多台服务器分发文件、执行命令 |
| 端口扫描 | 扫描服务器开放端口，识别服务类型 |
| AI 代码审查 | 读取本地代码，AI 自动审查并给修复建议 |

## 实现路线

### 第一阶段：技能管理器 + 存储
- 技能数据格式定义
- localStorage 读写封装
- 已安装技能加载/合并到 AI 上下文

### 第二阶段：技能商店 UI
- 商店主界面（分类 + 搜索 + 卡片）
- 安装确认弹窗（权限展示）
- 已安装/可更新管理

### 第三阶段：AI 自动生成技能
- AI 完成任务后的模式检测
- 技能草案生成
- 自动发现弹窗
- 用户确认后加入技能库

### 第四阶段：云端市场
- GitHub 仓库作为技能源
- 技能下载/签名验证
- 版本更新检查
- 社区提交流程

## 验证标准

1. 内置技能在首次打开商店时可见，可一键安装
2. 安装技能后 AI 对话中能调用新技能的工具
3. 卸载技能后对应工具不再可用
4. AI 完成任务后能正确弹窗提示新技能
5. 用户确认后新技能加入技能库
6. 签名验证：篡改后的技能文件安装时弹警告
