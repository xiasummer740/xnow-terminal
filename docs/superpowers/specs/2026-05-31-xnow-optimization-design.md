# XNOW 项目全面优化设计文档

**日期**: 2026-05-31  
**作者**: Claude (Kiro AI)  
**状态**: 已批准

## 1. 项目目标

将 electerm 分支版本彻底改造为 XNOW 品牌的专业终端工具，通过分阶段优化实现：
- Git 仓库干净整洁
- 品牌信息统一
- 代码结构清晰
- 文档完善
- 可维护性强

## 2. 整体架构

### 2.1 优化策略

采用**分阶段渐进式优化**策略，分为 4 个独立阶段：

```
阶段一：清理和提交 (30分钟)
    ↓
阶段二：品牌和依赖 (1-2小时)
    ↓
阶段三：代码重构 (2-3小时)
    ↓
阶段四：文档和测试 (1-2小时)
```

每个阶段完成后立即提交推送到 GitHub，确保多设备协作安全。

### 2.2 技术栈

保持现有技术栈不变：
- **桌面端**: Electron 41
- **前端**: React 19 + Vite 8 + Ant Design 6
- **状态管理**: Manate
- **后端**: Node.js + Express
- **测试**: Playwright (E2E) + Vitest (单元测试，新增)
- **代码规范**: Standard.js

### 2.3 风险控制

- 每个阶段独立验证（\`npm run build\` + \`npm run lint\`）
- 每个阶段独立提交，便于回滚
- 重构前先提交现有修改，避免混淆
- 保持主分支稳定，所有改动经过验证

├── utils/          # 通用工具函数
├── services/       # API 服务
├── schemas/        # 数据验证
